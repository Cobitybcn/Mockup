# Master Pattern: Workspace Layout

## Pattern

The application is organized as vertical worktables rather than separate administrative pages. Each workspace moves downward through orientation, a primary decision, visual browsing, active work, assignments, and results.

Horizontal galleries live inside this vertical flow. They support comparison without turning the workspace into a horizontal dashboard.

## Invariants

- Keep the workflow vertical.
- Begin with an editorial title and short description.
- Present the Primary Decision Block before secondary production stages when it is required.
- Use horizontal carousels for image browsing within the sequence.
- Place the active workspace before assignments and results.
- Keep imagery at the top of the visual hierarchy.
- Preserve generous white space, thin boundaries, and soft density.

## Never

- Convert the layout into a SaaS or KPI dashboard.
- Arrange equal-weight modules as a horizontal control center.
- Lead with forms, tables, metrics, or utilities.
- Fragment one production flow into unrelated cards.
- Reduce visual material to make room for unnecessary text.
- Introduce a new composition when an approved workspace flow already applies.
