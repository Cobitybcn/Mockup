# Master Pattern: Primary Actions

## Pattern

The Primary Action is the principal production commitment of a workspace stage. Important primary actions preferably use Decision Blocks instead of traditional SaaS buttons.

## Invariants

- Present one clear primary commitment per workspace stage.
- Prefer square Decision Block geometry for consequential actions.
- Use the established pigmentary pastel identity of the section.
- Give the action substantial presence and generous surrounding space.
- Use a short, action-oriented label.
- Keep secondary actions visually quieter.
- Show processing and results close to the action or affected workspace.

## Never

- Default to a traditional SaaS primary button for a major decision.
- Use saturated corporate button colors.
- Create several competing primary actions.
- Reduce the principal action to a minor toolbar control.
- Apply generic framework styling, gradients, heavy shadow, or glow.
