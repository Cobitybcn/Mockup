# Approved Visual Reference

## What pattern does this screen define?

Explore Scenes (`platform/mockup_combinations_review.php`) defines the approved Primary Action pattern: a single square, pastel dusty-rose block ("GENERATE THESE 4 VIEWS") placed beside the editorial header, with generous surrounding space and no competing action of equal weight anywhere else on the screen. It reads as a Decision Block used for the principal commitment of the workspace, not a traditional button. The rest of the screen (visual-direction carousel with a ring selection state, segmented camera-set choice, results grid below) is the same PASS-rated vertical workflow already covered by other references, shown here only for context.

## When should it be reused?

Reuse this exact treatment — square/near-square pastel block, short action-oriented label, one per workspace stage, positioned beside or after the header — as the default for any "principal commitment" action (generate, apply, publish, submit). It is the reference for `MASTER_PATTERNS/10_primary_actions`.

## What must never change?

- Only one Primary Action per workspace stage; never let a secondary action match its visual weight.
- Prefer the square/near-square Decision Block silhouette over a rectangular SaaS-style button for this role.
- Keep the label short and action-oriented ("Generate these 4 views"), never a long instruction.
- Give it generous surrounding space; never crowd it against other controls.
- Use the section's established pastel color, never a saturated corporate color.

## Known deviation — do not copy

Do not treat Mockup Lab's (`platform/mockup_variation_lab.php`) "Apply Changes" action or its permanently visible scale/rotate icon column as reference material. That screen is flagged `NEEDS CONSISTENCY PASS` in `design-system/audits/VISUAL_CONSISTENCY_MATRIX.md` for a generic primary action and permanent toolbars that should be folded — the opposite of the invariants in `MASTER_PATTERNS/10_primary_actions` and `MASTER_PATTERNS/09_workspace_panels`.
