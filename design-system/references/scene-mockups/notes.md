# Approved Visual Reference

## What pattern does this screen define?

Scene Mockups (`platform/mockup_combination_results.php`) defines the canonical Thumbnail Card + Glass Action pattern for a results board. Rectangular, image-led cards sit in a light grid with generous gutters. Each card carries only a favorite (top-left) and regenerate/delete (top-right) as small circular frosted-glass controls directly on the image, plus a compact scene/set badge and a short title below — no permanent action bar, no metadata block. The workspace header stays editorial (serif title, one short instruction line, one pastel Decision Block for the next primary action) and clearly subordinate to the image grid beneath it.

## When should it be reused?

Reuse this pattern for any results/library board where the primary content is a set of generated or curated images that the artist reviews, favorites, regenerates, or discards — e.g. Mockup Album, Video results, Scene Studio library. It is the reference to check against whenever a screen is flagged in the audit for reintroducing toolbars below thumbnails, oversized metadata, or generic buttons instead of Glass Actions.

## What must never change?

- Rectangular Thumbnail Card geometry with the image occupying nearly the full card.
- Glass Actions live on the image (favorite top-left, secondary actions top-right); never a toolbar row beneath the thumbnail.
- One pastel Decision Block for the primary commitment ("Explore New Scenes"), never a rectangular SaaS button.
- Compact, muted badges (scene/set) and a short title — never expanded metadata that competes with the image.
- Generous white space between cards; the grid must not be compressed to fit more items per row.
