# Artist Site Manager Visual Language

## Purpose

Artist Site Manager is the operational bridge between Artwork Mockups and the public Artist Website. It is a separate workspace with shared family traits, not a copy of either surface.

## Character

The manager combines editorial calm with operational clarity. It supports review, publication, prints, stock, orders, domains, and site configuration without becoming a generic SaaS dashboard.

## Hierarchy

1. The selected artwork, series, note, or print.
2. The active management workspace.
3. Section title and immediate state.
4. One principal action.
5. Supporting settings and history.

## Composition

- A compact, quiet header identifies the artist site, contains the four primary text tabs, and provides return links.
- Secondary navigation uses the same restrained text-and-underline pattern.
- Visual entities use compact horizontal image rails sized for efficient work on a 15-inch laptop.
- Editors use broad panels with comfortable fields.
- Secondary settings are folded by default.
- Orders may use a calm table because they are genuinely tabular.

## Color and Type

- Warm neutral surfaces and fine borders connect the manager to the product family.
- Editorial serif headings provide continuity without imitating the public website.
- Readable sans-serif text carries operational information.
- Dusty rose, sage, ochre, and muted lilac identify states or areas only when needed.
- The principal action uses a soft pastel surface and short label.

## Interaction

- Selecting an image opens its work area in place.
- Feedback appears next to the action that produced it.
- Publishing is explicit; writing does not update the public site immediately.
- Content synchronization never overwrites commerce data.
- Destructive actions remain secondary and require confirmation.

## Content Order

The public website keeps its protected order:

```text
Artworks · Series · Constellations · Studio Notes · Artist · Inquire
```

Within Site Manager, Constellations is a single optional country field on each artwork rather than a separate administrative section. Store contains Prints and Orders. Settings contains Site, Domain, Payments, and Shipping.
